Railway / Production Deployment Checklist

1) Required environment variables

- `NODE_ENV=production`
- `FRONTEND_URL=https://aientity.co.uk` (set this to your exact frontend origin)
- `COOKIE_DOMAIN=.aientity.co.uk` (optional — enables cookie for subdomains)
- `FRONTEND_ORIGINS` (optional, comma-separated extra allowed frontend origins)
- `RAILWAY_STATIC_URL` (if using Railway static deploy; optional)

2) Cookie settings

- The server will set authentication cookies with `SameSite=None; Secure` when `NODE_ENV=production`.
- Ensure your backend is served over HTTPS so browsers accept `Secure` cookies.

3) Railway dashboard steps

- In Railway, go to your project > Variables, and add the variables listed above.
- Set `NODE_ENV` to `production`.
- Add `FRONTEND_URL` with the value `https://aientity.co.uk` (or your domain).
- (Optional) Add `COOKIE_DOMAIN` set to `.aientity.co.uk` if you need cookie on subdomains.
- Redeploy the service after updating variables.

4) Verify after deploy

- Confirm the app is reachable at your domain (https).
- From the browser's DevTools (Application > Storage > Cookies), perform a login/register and confirm:
  - A cookie named `token` is present for your domain
  - The cookie has `SameSite=None` and `Secure` flags

- Quick HTTP test (replace host if needed):

  Using native curl on Windows (powershell):

```powershell
curl.exe -i -X POST "https://aientity.co.uk/api/auth/register" -H "Content-Type: application/json" -d "{\"email\":\"test+deploy@example.com\",\"password\":\"password123\"}"
```

- The response should include a `Set-Cookie` header similar to:

```
Set-Cookie: token=...; Max-Age=604800; Path=/; Expires=...; HttpOnly; Secure; SameSite=None
```

5) Troubleshooting

- If cookies are not set in the browser:
  - Confirm the response `Set-Cookie` header includes `SameSite=None; Secure`.
  - Confirm the site is served over HTTPS.
  - Check browser console/network for CORS errors — ensure your frontend origin is listed in `FRONTEND_URL` or `FRONTEND_ORIGINS`.

6) Rollback and notes

- If you need to roll back, revert the environment variable changes and redeploy the previous release.
- The codebase includes a fallback to `https://aientity.co.uk` when in production and no frontend origin is configured; set `FRONTEND_URL` to avoid relying on this fallback.

7) Optional: Cookie domain across subdomains

- To allow cookies on all subdomains (eg `app.aientity.co.uk` and `www.aientity.co.uk`), set `COOKIE_DOMAIN` to `.aientity.co.uk` in Railway variables.
